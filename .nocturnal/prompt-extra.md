## Commit Message Format

Use this format for all commits:

- Start with a verb in imperative mood (e.g., "Add", "Fix", "Remove", "Update")
- No prefix or scope (no "web:", "td:", "GitHub:" etc.)
- Lowercase after the first word
- No period at the end
- Keep the subject line concise (under 72 characters)

Examples:
- Fix unresolved comments detection for GitHub PRs
- Add copy button for task ID on issue detail page
- Remove stale worktrees and locks on garbage collection
- Display PR/MR link on issue detail page
